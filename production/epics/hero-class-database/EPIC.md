## Overview

`HeroClassDatabase` is a typed accessor over the read-only `HeroClass` resources
loaded by `DataRegistry` at boot. It provides per-stat-per-level helpers
(`stat_at_level`, `hero_tick_output`), counter-archetype matchup checks
(`is_class_counter`), recruitable filters (tier==1 in MVP), and a stable
id→resource lookup that survives save/restore via Godot's resource cache.

Class data is immutable at runtime; mutable hero state lives in HeroRoster
(Feature layer). The `EnemyArchetypes` constant set authored here is the
single source of truth for archetype tags consumed by EnemyDatabase and
ClassEnemyMatchupResolver.

## Cross-Cutting Inputs (received from)

- `DataRegistry` (Foundation) — resolves `HeroClass` resources by id

## Cross-Cutting Outputs (consumed by)

- EnemyDatabase (Core, this layer) — reads `EnemyArchetypes` constants
- ClassEnemyMatchupResolver (Feature) — reads `is_class_counter()`
- HeroRoster (Feature) — reads `HeroClass` ref by id, holds `instance_id`
- Recruitment / HeroLeveling (Feature) — read `tier` for cost curves
- Combat resolver (Feature) — reads `stat_at_level()`, `hero_tick_output()`
