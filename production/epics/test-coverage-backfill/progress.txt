## Codebase Patterns

- Test runner invocation (macOS, Godot mono build):
    /Applications/Godot_mono.app/Contents/MacOS/Godot --headless \
      -s addons/gdUnit4/bin/GdUnitCmdTool.gd --ignoreHeadlessMode \
      -a tests/unit/<dir-or-file>
  The wrapper `tests/gdunit4_runner.gd` exists but the direct CmdTool form is
  what CI uses. `--ignoreHeadlessMode` is REQUIRED — without it the runner
  exits 103 (headless-mode unsupported guard).
- Audit predicate (per EPIC technical considerations):
    grep -nE "^func [a-z]" src/core/<system>/<file>.gd   # public funcs
    grep -rn "<func_name>" tests/                         # references
- Pre-existing baseline failures observed (NOT regressions, do not let them
  block US-NNN closure if your file is unaffected):

      (2 fail count — single test, recorded twice; investigation deferred,
      not in scope for test-coverage-backfill epic per FR-8 reading).

  and `test_<func>_<scenario>_<expected>`. See
  tests/unit/audio_router/volume_persistence_round_trip_test.gd for the

**Learnings for future iterations:**
- For accessor-style functions (`set_*` DI seams) the canonical happy+edge pair
  is "replaces_internal_field" (happy) + "survives_ready_call" (edge —
  injection-before-_ready survives the lazy-default branch). When auditing
  other autoloads' DI seams (TickSystem, SaveLoadSystem, SceneManager), look
  for this paired pattern in autoload_skeleton_*.gd files and accept it as
  full coverage.
- For functions with implementation branches gated by data-registry state
  (e.g., `resolve_floor_by_snapshot_id` only returns non-null if "floors" were
  a registered category — it isn't), BOTH observable-behavior branches (empty
  + unknown) ARE the full coverage surface. A "happy path" doesn't have to
  return a non-null/non-error value; it just has to assert the documented
  return shape. This is consistent with FR-2 ("documented return shape /
  behavior").
- Heavy-coverage files like dungeon_run_orchestrator (42 test files) only
  need an audit pass — searching `grep -rn "<func_name>" tests/` and
  spot-checking the assertion shape suffices. Don't be tempted to "improve"
  test names or refactor existing tests; that's out of scope per FR-8 spirit
  ("non-regression in baseline" implies "don't touch passing tests").

  "Production callers MUST NOT set this"); coverage via every
  compute_offline_batch happy test counts because the seam IS the input
  channel for those tests. Don't add a separate dedicated test for DI seams
  whose only legitimate caller is the test harness itself.

  asserting the *cast/filter behavior* (returns null/empty on miss; returns
  typed result + sort order on hit), not by re-asserting DataRegistry's
