## Codebase Patterns

- Test runner invocation (macOS, Godot mono build):
    /Applications/Godot_mono.app/Contents/MacOS/Godot --headless \
      -s addons/gdUnit4/bin/GdUnitCmdTool.gd --ignoreHeadlessMode \
      -a tests/unit/<dir-or-file>
  The wrapper `tests/gdunit4_runner.gd` exists but the direct CmdTool form is
  what CI uses. `--ignoreHeadlessMode` is REQUIRED — without it the runner
  exits 103 (headless-mode unsupported guard).
- Audit predicate (per EPIC technical considerations):
    grep -nE "^func [a-z]" src/core/<system>/<file>.gd   # public funcs
    grep -rn "<func_name>" tests/                         # references
- Pre-existing baseline failures observed (NOT regressions, do not let them
  block US-NNN closure if your file is unaffected):

      (2 fail count — single test, recorded twice; investigation deferred,
      not in scope for test-coverage-backfill epic per FR-8 reading).
- Test naming convention in this directory permits both `test_<system>_<...>`
  and `test_<func>_<scenario>_<expected>`. See
