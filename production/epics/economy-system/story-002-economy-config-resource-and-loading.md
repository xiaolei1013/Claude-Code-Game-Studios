> **Epic**: economy-system
> **Status**: Complete
> **Layer**: Core
> **Type**: Config/Data
> **Manifest Version**: 2026-04-24

## Context

**GDD**: `design/gdd/economy-system.md`
**Requirements**: TR-economy-006, TR-economy-007, TR-economy-008, TR-economy-009, TR-economy-010 (tables + constants)
*(Full requirement text lives in `docs/architecture/tr-registry.yaml`.)*

**Governing ADR(s)**: ADR-0011 (Resource Schemas Core Databases — `EconomyConfig extends GameData` pattern), ADR-0006 (DataRegistry boot scan + `resolve("config", "economy_config")`), ADR-0013 (single-source-of-truth tuning knobs)
**ADR Decision Summary**: All 26 tuning knobs in GDD §G live in `assets/data/config/economy_config.tres` (`EconomyConfig extends GameData` per ADR-0011). DataRegistry rank-1 boot scan resolves the resource; the loader fails ERROR-state if the resource is missing or malformed.

**Engine**: Godot 4.6 | **Risk**: LOW
**Engine Notes**: `Resource` subclass with `@export` properties is stable ≥ Godot 4.0; `Dictionary[int, int]` typed-container syntax is post-cutoff but precedent-verified.

**Control Manifest Rules (Core Layer, Economy)**:
- **Required**: All 26 tuning knobs in `assets/data/config/economy_config.tres`; no hardcoded balance values in `economy.gd`. — ADR-0013
- **Required**: `EconomyConfig extends GameData` pattern (ADR-0011 schema discipline).
- **Allowlisted**: structural constants in `economy.gd` are `GOLD_SANITY_CAP = 1_000_000_000_000` and `OFFLINE_REPLAY_REASON = "offline_replay"` only. — ADR-0013
- **Forbidden**: hardcoded balance literal anywhere outside `economy_config.gd` (full grep enforcement lands in Story 013).

## Acceptance Criteria

- [ ] TR-economy-006: `BASE_DRIP` table on EconomyConfig: `Array[int]` size 5 with values `[2, 4, 7, 12, 8]` (1-indexed via `BASE_DRIP[floor_index - 1]` in usage)
- [ ] TR-economy-006: `MATCHUP_DRIP_BONUS: float = 1.0` (default; per-class overrides not in MVP)
- [ ] TR-economy-007: `BASE_KILL` table: `Dictionary[int, int]` keyed by tier `{1: 10, 2: 35, 3: 80}`; `MATCHUP_GOLD_MULTIPLIER: float = 1.5`
- [ ] TR-economy-008: `BASE_RECRUIT` table: `Dictionary[int, int]` keyed by tier; `RECRUIT_RATIO: float = 1.8`
- [ ] TR-economy-009: `BASE_LEVEL` table: `Dictionary[int, int]` keyed by tier; `LEVEL_RATIO: float = 1.6`; `LEVEL_CAP: int = 15`
- [ ] TR-economy-010: `FLOOR_CLEAR_BONUS` table: `Dictionary[int, int]` `{1: 500, 2: 1200, 3: 3000, 4: 7500, 5: 18000}`; 1-based indexing
- [ ] All 26 GDD §G knobs present as `@export`-annotated properties on `EconomyConfig`
- [ ] `assets/data/config/economy_config.tres` exists and is loadable via `DataRegistry.resolve("config", "economy_config")` returning a non-null `EconomyConfig`
- [ ] Boot validation: missing resource OR any required-field empty triggers DataRegistry ERROR state with `push_error("EconomyConfig invalid: [field]")`
- [ ] Load-time schema validation rejects: any `BASE_DRIP` length ≠ 5, any negative table value, `MATCHUP_GOLD_MULTIPLIER ≤ 0.0`, `RECRUIT_RATIO ≤ 1.0`, `LEVEL_CAP < 1`

## Implementation Notes

*Derived from ADR-0011 §Decision (resource schema pattern) + ADR-0013 §G+§Decision (tuning-knob enumeration):*

- Create `src/core/economy/economy_config.gd` with `class_name EconomyConfig extends GameData` (GameData defined per ADR-0011 in DataRegistry epic — Story S1-M4 already landed in Sprint 1).
- Each of the 26 GDD §G knobs becomes an `@export var` with appropriate type. Use `@export_range` where the GDD §G safe-range column specifies a min/max.
- Write `assets/data/config/economy_config.tres` once with all 26 knobs populated to GDD §D defaults. The `.tres` should NOT be regenerated by code; it is hand-authored / inspector-edited.
- Hook into DataRegistry per ADR-0006: file is discovered automatically by the boot scan since it lives under `assets/data/config/`. No additional registration needed if DataRegistry's category mapping recognizes `config` directory (verify against data-registry Story-005 implementation).
- Validation: implement `EconomyConfig._validate() -> Array[String]` returning a list of error strings; called by DataRegistry per-type validator (per data-registry epic Story 005). An empty array means OK.
- The `EconomyConfig._validate` should enforce all 7 schema constraints listed in the AC. Below MIN_CONTENT_COUNT triggers ERROR per ADR-0006 boot policy.

## QA Test Cases

- **TR-economy-006/007/008/009/010**: Constants present + correct shape
  - **Given**: `assets/data/config/economy_config.tres` exists at engine boot
  - **When**: `DataRegistry.resolve("config", "economy_config")` is called after DataRegistry reaches READY state
  - **Then**: returns `EconomyConfig` with: `BASE_DRIP == [2, 4, 7, 12, 8]`; `BASE_KILL == {1: 10, 2: 35, 3: 80}`; `RECRUIT_RATIO == 1.8`; `LEVEL_RATIO == 1.6`; `LEVEL_CAP == 15`; `FLOOR_CLEAR_BONUS == {1: 500, 2: 1200, 3: 3000, 4: 7500, 5: 18000}`; `MATCHUP_GOLD_MULTIPLIER == 1.5`; `MATCHUP_DRIP_BONUS == 1.0`
  - **Edge cases**: each constant verified independently (one assert per constant for failure-isolation)

- **Schema validation**: invalid `.tres` rejected at boot
  - **Given**: a corrupted test `.tres` fixture with `BASE_DRIP` length 4
  - **When**: DataRegistry boot scan loads it
  - **Then**: `_validate()` returns a non-empty error list; `push_error` fires; DataRegistry transitions to ERROR state below MIN_CONTENT_COUNT
  - **Edge cases**: also test `RECRUIT_RATIO = 1.0` (boundary), `LEVEL_CAP = 0`, negative table value — each must fail validation

- **All 26 knobs present**
  - **Given**: EconomyConfig instance loaded
  - **When**: introspect `get_property_list()` filtered to `@export` properties
  - **Then**: 26 distinct knob properties enumerated, each non-default-empty (sanity check, not value-correctness — that's other tests)
  - **Edge cases**: documenting which knobs map to which GDD §G rows; failure-mode is "GDD updated but config schema drifted"
