## Overview

`EnemyDatabase` is a typed accessor over read-only `EnemyData` resources loaded
by `DataRegistry` from `assets/data/enemies/`. Enemies are static data: no
leveling, no per-enemy loot override, no resistances in MVP. Schema validation
enforces archetype-distribution invariants (Tier-1/Tier-2 contain one of each
MVP archetype; Tier-3 all bruiser; exactly one boss = Ancient Rootking) and
HP-band ranges per tier.

Combat reads HP/attack/speed directly from `EnemyData`; the orchestrator reads
`is_boss` to trigger boss-death fanfare and floor-clear bonus. Kill gold is
computed by Economy via `BASE_KILL[tier] * matchup_multiplier`.

## Cross-Cutting Inputs (received from)

- `HeroClassDatabase` (Core, sibling epic) — `EnemyArchetypes` constants
- `DataRegistry` (Foundation) — resolves `EnemyData` resources by id

## Cross-Cutting Outputs (consumed by)

- BiomeDungeonDatabase (Core, sibling epic) — `Floor.enemy_list[].enemy_id` must resolve
- CombatResolver (Feature) — reads HP/attack/speed
- DungeonRunOrchestrator (Feature) — reads `is_boss`; emits `enemy_killed`
- ClassEnemyMatchupResolver (Feature) — reads archetype for matchup math
